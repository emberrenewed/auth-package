1. Big picture ١. وێنەی گشتی

This project is not a full Laravel app. It is a reusable package (library) named emberrenewed/laravel-auth-kit. Your real app installs it with Composer. Code lives under namespace Technobase\AuthKit. ئەم پڕۆژەیە ئەپی تەواوی Laravel نییە. پاکێجێکی دووبارە بەکارھێنراوە (لایبرەری) بە ناوی emberrenewed/laravel-auth-kit. ئەپی ڕاستەقینەت بە Composer دایەدەگرێت. کۆدەکە لە ژێر namespaceـی Technobase\AuthKitـدایە.
Think of it like a ready-made authentication engine: password, Google, Facebook, GitHub, Email OTP, WhatsApp OTP — for both browser sessions and API tokens (Sanctum). وەک مۆتۆڕێکی ئامادەی ناسنامە / چوونەژوورەوە بیربکەرەوە: وشەی نهێنی، Google، Facebook، GitHub، OTPـی ئیمەیڵ، OTPـی WhatsApp — بۆ ھەردووک سێشنی وێبگەڕ و توکنی APIـی Sanctum.

2. Folder tree (map) ٢. دارەی فۆڵدەر (نەخشە)

Green = folders that hold the brain of the package. Blue = files you touch often. Amber = supporting config/DB/tests. سەوز = فۆڵدەرەکانی مێشکی پاکێج. شین = فایلەکانی زۆر دەستکاری دەکرێن. زەرد = ڕێکخستن / داتابەیس / تێست.

├── config/

│   └── web.php            # /auth/…
├── database/migrations/   # OTP table + user columns

├── postman/               # API collection for Postman
└── src/                   # ← ALL PHP business logic
    ├── AuthKitServiceProvider.php
    ├── Console/            # artisan auth-kit:install

3. Each folder — purpose, why, name ٣. ھەر فۆڵدەرێک — مەبەست، ھۆکار، ناو

src/ core

Purpose

Holds all PHP classes of the package. This is the brain.

ھەموو کلاسەکانی PHPـی پاکێج لێرەیە. ئەمە مێشکی پاکێجە.

Why name

src = source (source code). Industry standard for “where the real code lives”.

src واتە source (سەرچاوەی کۆد). ستانداردی پیشەسازییە بۆ «کۆدی سەرەکی لێرەیە».

src/Contracts/

Purpose

Interfaces only: AuthDriver, CredentialIssuer, SubjectResolver, OtpChannel. They define what a class must do, not how.

تەنھا interface: پێناسە دەکەن کە کلاسێک چی دەبێت بیکات، نەك چۆن.

Why name

Contract = a written promise between pieces of code. Like a hiring contract: “you must implement these methods.”

Contract واتە گرێبەست/بەڵێننامە. وەک گرێبەستی کار: «دەبێت ئەم مێتۆدانە جێبەجێ بکەیت.»

Why use

So you can add WhatsApp OTP or a new social login without rewriting the AuthenticateAction — just a new Driver that obeys the same AuthDriver contract.

بۆ ئەوەی OTPـی نوێ یان loginـی کۆمەڵایەتی نوێ زیاد بکەیت بەبێ گۆڕینی AuthenticateAction — تەنھا Driverـێکی نوێ کە ھەمان گرێبەستی AuthDriver جێبەجێ دەکات.

src/Drivers/

Purpose

One class per login method: Password, Google, Facebook, GitHub, EmailOtp, WhatsAppOtp. Each validates input and builds a normalized identity (email/phone/provider ids).

یەک کلاس بۆ ھەر شێوازی چوونەژوورەوە. ھەر کامێک داواکاری پشتڕاست دەکاتەوە و ناسنامەیەکی یەکگرتوو دروست دەکات (ئیمەیڵ/تەلەفۆن/provider).

Why name

Driver = like a car driver for a specific road. PasswordDriver “drives” password login; GoogleDriver “drives” Google login. Same steering wheel (AuthDriver), different roads.

Driver وەک شۆفێر بۆ ڕێگایەکی دیاریکراو. PasswordDriver شۆفێری وشەی نهێنییە؛ GoogleDriver شۆفێری Googleـە. ھەمان ئامێری لێخوڕین، ڕێگای جیاواز.

src/Http/

Purpose

Everything that talks to the HTTP world: controllers, form validation, JSON resources, callbacks that return responses.

ھەموو شتێک کە لەگەڵ جیھانی HTTP قسە دەکات: کۆنترۆڵەر، پشتڕاستکردنەوەی فۆرم، JSON، callbackـەکان کە وەڵام دەگەڕێننەوە.

Why name

Http = HyperText Transfer Protocol. Laravel convention: put request/response code under Http.

Http پڕۆتۆکۆلی وێبە. لە Laravelـدا کۆدی داواکاری/وەڵام لەژێر Http دەخرێت.

src/Http/Controllers/ Api/ + Web/

Purpose

Thin doors. Routes call controller methods. Controllers almost immediately hand work to Actions. Api returns JSON/token; Web returns redirects/session.

دەرگایەکی تەنک. ڕوتەکان کۆنترۆڵەر بانگ دەکەن. کۆنترۆڵەر خێرا کار دەدات بە Action. Api JSON/توکن دەگەڕێنێتەوە؛ Web ڕیدایرێکت/سێشن.

Why name

Controller = control tower: receives the plane (request), decides which runway (callback). Api = Application Programming Interface (apps/mobile). Web = browser pages.

Controller وەک تاوەری کۆنترۆڵ: فڕۆکە (داواکاری) وەردەگرێت، ڕێگا دیاری دەکات. Api بۆ ئەپ/ مۆبایل. Web بۆ وێبگەڕ.

src/Http/Actions/

Purpose

Where the real work lives: AuthenticateAction, LogoutAction, SendOtpAction, ForgotPasswordAction, ResetPasswordAction, ListProvidersAction, RedirectToProviderAction. Invokable classes (__invoke) called from AuthController.

شوێنی کرداری ڕاستەقینە: Authenticate، Logout، SendOtp… کلاسە Invokableـەکان کە AuthController بانگیان دەکات.

Why name

Action = one clear job. Laravel pattern: thin controller → action class does the work.

Action واتە یەک کارێکی دیاریکراو. شێوازی Laravel: کۆنترۆڵەری تەنک → کلاسە Action کارەکە دەکات.

src/Http/CredentialIssuers/

Purpose

After login succeeds: SanctumCredentialIssuer creates an API token; SessionCredentialIssuer logs the user into a web session.

دوای سەرکەوتنی چوونەژوورەوە: یەکێک توکنی API دروست دەکات؛ ئەوی دیکە سێشنی وێب دەکاتەوە.

Why name

Credential = proof you are logged in (token or session). Issuer = the one who issues/gives that proof (like a passport office).

Credential = بەڵگەنامەی چوونەژوورەوە (توکن یان سێشن). Issuer = دەستگای دەرکردنی ئەو بەڵگەیە (وەک ئۆفیسی پاسپۆرت).

src/Http/Requests/

Purpose

Form Request classes: rules for login, social token, forgot/reset password — reject bad input early.

یاساکانی پشتڕاستکردنەوەی فۆرم — داواکاری خراپ زوو ڕەت دەکرێتەوە.

Why name

Laravel name for “validated HTTP request object.”

ناوی Laravel بۆ «داواکاری HTTPـی پشتڕاستکراو.»

src/Support/

Purpose

DriverRegistry = map of name → driver. EloquentSubjectResolver = find/create User. NormalizedIdentity = same identity shape for every driver.

تۆماری درایڤەرەکان، دۆزینەوە/دروستکردنی User، و شکڵێکی یەکگرتووی ناسنامە بۆ ھەموو درایڤەرەکان.

Why name

Support = helpers that support the main flow, not the public HTTP face.

Support = یارمەتیدەرەکان کە پشتگیری ڕێڕەوی سەرەکی دەکەن، نەك ڕووی گشتی HTTP.

src/Otp/ + Channels/

Purpose

OtpManager creates, stores, verifies one-time codes. Channels send them (Mail, WhatsApp Cloud, Log for testing).

کۆدی یەکجار بەکارهێنان دروست/پاشەکەوت/پشتڕاست دەکات. Channelـەکان دەینێرن (ئیمەیڵ، واتساپ، یان Log بۆ تێست).

Why name

OTP = One-Time Password. Channel = the pipe/path the message travels through.

OTP = وشەی نهێنی یەکجار بەکارھێنان. Channel = کەناڵ/بۆری ناردنی پەیام.

src/Events/ · src/Exceptions/

Purpose

Events announce something happened (LoginSucceeded) so your app can listen (send mail, log). Exceptions are typed failures (banned user, bad credentials, unknown driver).

ڕووداوەکان ئاگادار دەکەنەوە کە شتێک ڕوویدا (بۆ گوێگرتن لە ئەپەکەت). Exceptionـەکان ھەڵەی جۆردارن (بان، وشەی نهێنی ھەڵە، درایڤەری نەناسراو).

Why name

Event = something that happened. Exception = an exceptional (abnormal) situation that interrupts the happy path.

Event = ڕووداو. Exception = دۆخی نائاسایی کە ڕێڕەوی ئاسایی دەپسێنێت.

routes/ · config/ · database/ · tests/

Purpose

routes map URLs → controllers. config turns features on/off (which drivers, prefixes). database/migrations add OTP table + columns like phone. tests prove the flow still works.

routes URL دەبەستێتەوە بە کۆنترۆڵەر. config تایبەتمەندی دادەسەرەخات. migrations خشتە/ستوونی داتابەیس زیاد دەکەن. tests دڵنیایی کە ڕێڕەو ھێشتا کار دەکات.

4. How login works (full flow) ٤. چۆن چوونەژوورەوە کاردەکات (ڕێڕەوی تەواو)

Example: POST /api/auth/login with email + password. Same pipeline is used for Google/OTP — only the Driver changes. نموونە: POST /api/auth/login بە ئیمەیڵ + وشەی نهێنی. ھەمان ھێڵ بۆ Google/OTP بەکاردێت — تەنھا Driver دەگۆڕێت.
1
Request hits a route داواکاری دەگاتە ڕووت

URL is defined in routes/api.php or routes/web.php. Prefix comes from config/auth-kit.php.

URL لە ڕووتدا دیاریکراوە. پێشگر (prefix) لە کۆنفیگەوە دێت.

2
Controller opens the door کۆنترۆڵەر دەرگا دەکاتەوە

Http/Controllers/Api/AuthController method login() — thin. It calls the AuthenticateAction action.

مێتۆدی login تەنکە؛ راستەوخۆ AuthenticateAction بانگ دەکات.

3
Action resolves the Driver Action درایڤەر دەدۆزێتەوە

Http/Actions/AuthenticateAction asks Support/DriverRegistry for e.g. password. Checks config that this driver is allowed for api or web.

لە DriverRegistry درایڤەر وەردەگرێت (وەک password) و سەیری کۆنفیگ دەکات کە بۆ api/web ڕێگەپێدراوە.

4
Driver validates & builds identity درایڤەر پشتڕاست دەکاتەوە و ناسنامە دروست دەکات

Drivers/PasswordDriver (or Google/OTP…) runs validate() then resolveIdentity() → Support/NormalizedIdentity. Event: LoginAttempted.

درایڤەر validate و resolveIdentity دەکات → NormalizedIdentity. ڕووداوی LoginAttempted دەنێردرێت.

5
Find (or create) the Subject = User دۆزینەوە (یان دروستکردن)ی Subject = بەکارھێنەر

Support/EloquentSubjectResolver looks up the user by email/phone/provider. If banned → fail. Name subject means “the person being authenticated.”

EloquentSubjectResolver بەکارھێنەر بە ئیمەیڵ/تەلەفۆن/provider دەدۆزێتەوە. ئەگەر بان بێت → شکست. subject واتە «ئەو کەسەی ناسنامەی بۆ پشتڕاست دەکرێتەوە.»

6
Issue credentials (token or session) دەرکردنی بەڵگەنامە (توکن یان سێشن)

API → CredentialIssuers/SanctumCredentialIssuer. Web → SessionCredentialIssuer. Then LoginSucceeded event. Response via Http/Resources/AuthResource.

API → Sanctum توکن. Web → سێشن. پاشان ڕووداوی LoginSucceeded و وەڵام لە ڕێگەی AuthResource.

OTP extra steps: first SendOtpAction + Otp/OtpManager + channel send the code; later verify uses email_otp / whatsapp_otp driver inside the same AuthenticateAction pipeline. ھەنگاوی زیادەی OTP: سەرەتا SendOtp + OtpManager + کەناڵ کۆد دەنێرن؛ دواتر verify ھەمان AuthenticateAction بەکاردێنێت بە درایڤەری email_otp / whatsapp_otp.

5. API vs Web — same engine, different door ٥. API بەرامبەر وێب — ھەمان مۆتۆڕ، دەرگای جیاواز

API (mobile / SPA) API (مۆبایل / SPA)

Routes in routes/api.php → usually under /api/auth/…. Returns JSON + Sanctum bearer token. Good for Flutter, React Native, Postman.

ڕووتەکانی api → زۆرجار /api/auth/…. JSON + توکنی Sanctum دەگەڕێنێتەوە. باشە بۆ Flutter، React Native، Postman.

Web (browser) وێب (وێبگەڕ)

Routes in routes/web.php → /auth/…. Cookie session + redirects (especially OAuth Google/Facebook/GitHub redirect → callback).

ڕووتەکانی web → /auth/…. سێشنی کووکی + ڕیدایرێکت (بەتایبەتی OAuth: ڕەوانەکردن بۆ Google پاشان گەڕانەوە).

6. Name glossary (perfect meaning) ٦. فەرھەنگی ناوەکان (مانای تەواو)

Auth Kit

Auth = Authentication (proving who you are). Kit = toolbox / ready set of tools. So: a ready toolbox for login.

Auth = ناسنامە پشتڕاستکردنەوە (سەلماندنی تۆ کێیت). Kit = سندوقی ئامراز / کۆمەڵە ئامادە. واتە: سندوقێکی ئامادە بۆ چوونەژوورەوە.

Subject

In security English, the subject is the entity being authenticated (usually your User model). Not “school subject” — “the person under examination.”

لە زمانی ئاسایشدا، subject ئەو بونەوەرەیە کە ناسنامەی بۆ پشتڕاست دەکرێتەوە (زۆرجار مۆدێلی User). نەك «بابەتی قوتابخانە» — بەڵکو «ئەو کەسەی پشکنین دەکرێت.»

Flavor (api / web)

Internal name for “which taste of auth”: API flavor issues tokens; Web flavor issues sessions. Same recipe, different serving.

ناوی ناوخۆیی بۆ «چ جۆرێکی ئۆس»: flavorـی API توکن دەدات؛ flavorـی Web سێشن دەدات. ھەمان ڕێچکە، جۆری جیاوازی خزمەتکردن.

NormalizedIdentity

Normalize = make same shape. Google, password, and OTP all produce different raw data; this class unifies them into one simple identity object.

Normalize = کردن بە ھەمان شکڵ. Google و وشەی نهێنی و OTP داتای جیاواز دەدەن؛ ئەم کلاسە یەکیان دەکات بۆ یەک تەنۆکەی ناسنامە.

Registry

DriverRegistry is a phone book: name "google" → GoogleDriver class. Register = put in the book; resolve = look up.

DriverRegistry وەک پەیوەندینامە/تەلەفۆنبووک: ناوی "google" → کلاسەکەی GoogleDriver. تۆمارکردن = دانان لە کتێب؛ resolve = گەڕان.

ServiceProvider

Laravel’s boot loader for the package: registers drivers, loads migrations, publishes config, registers auth-kit:install.

دەستپێکەری Laravel بۆ پاکێج: درایڤەرەکان تۆمار دەکات، migration بار دەکات، کۆنفیگ بڵاو دەکاتەوە، فەرمانی install تۆمار دەکات.
